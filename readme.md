# 接口列表

## 1.健康检查接口

检查服务是否正常运行

- **URL**:`/health`
- **方法**:`GET`

### 响应示例

```json
{
    "state":"healthy"
}
```



# 2.任务创建接口

创建任务

- **URL**:`/api/v1/task/create`
- **方法**:`POST`

### 请求参数

| 参数名  | 类型          | 必须 | 描述             |
| ------- | ------------- | ---- | ---------------- |
| task_id | String        | 否   | 查询任务进展id   |
| quality | List[Quality] | 是   | 待分析的任务文本 |

#### Quality参数

| 参数名  | 类型   | 必须 | 描述             |
| ------- | ------ | ---- | ---------------- |
| iden    | String | 是   | 单条分析文本的id |
| quality | String | 是   | 单条分析文本     |

### 返回参数

| 参数名  | 类型   | 返回条件 | 描述               |
| ------- | ------ | -------- | ------------------ |
| task_id | String | 必定返回 | 用于查询任务进展id |

### 描述

1. 当`task_id`未传入时，默认为新任务，将作为一个独立的任务进行执行。
2. 当`task_id`已传入时，会查询`task_id`是否存在，如果不存在，默认为新任务，将作为一个独立的任务进行执行。
3. 当`task_id`已传入时，会查询`task_id`是否存在，如果已存在，尝试加入已有的任务进行执行，如果已有的任务未执行完成则返回400，并拒绝执行。
4. 当`task_id`已传入时，会查询`task_id`是否存在，如果已存在，尝试加入已有的任务进行执行，如果已有的任务执行完成则加入到该任务的数据中进行执行。

## 3.任务查询接口

查询任务

- **URL**:`/api/v1/task/query/<task_id>?immediately=`
- **方法**:`GET`

### 请求参数

| 参数名      | 类型   | 必须 | 描述               |
| ----------- | ------ | ---- | ------------------ |
| task_id     | String | 是   | 用于查询任务进展id |
| immediately | Bool   | 否   | 是否返回结果       |

### 返回参数

| 参数名      | 类型              | 返回条件              | 描述                                           |
| ----------- | ----------------- | --------------------- | ---------------------------------------------- |
| error       | String            | 必定返回              | 发生的错误信息，没有错误则返回”“               |
| state       | String            | 必定返回              | 任务状态，当完成时值为finished，有多个中间状态 |
| total       | Int               | 必定返回              | 该任务的数据总数                               |
| current     | Int               | 必定返回              | 当前中间状态已经完成的数目                     |
| used_time   | float             | 必定返回              | 当前中间状态已经耗时                           |
| total_time  | float             | 必定返回              | 当前中间状态可能需要执行的总时间               |
| remain_time | float             | 必定返回              | 当前状态可能需要执行的剩余时间                 |
| quality     | list[QualityTask] | 当immediately为True时 | 返回数据较多，建议当结果为finished时再取结果   |

#### QualityTask参数

| 参数名        | 类型   | 描述                       |
| ------------- | ------ | -------------------------- |
| quality       | String | 偏差文本                   |
| subject       | String | 偏差主体                   |
| status        | String | 偏差状态                   |
| classify      | int    | 偏差分类，为-1时表示未分类 |
| classify_name | String | 偏差分类名                 |
| iden          | String | 偏差文本id                 |
